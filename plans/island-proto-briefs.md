# Island prototype design briefs (research output, 2026-06-18)
Source: `island-design-research` workflow (Opus). 5 visual-only mockup directions for the BlitzOS dynamic island. Build-ready; each proto = `src/renderer/src/notch/protos/<id>.tsx`.

## apple — True Pill
**Philosophy:** Pixel-accurate iOS Dynamic Island ported to macOS: the shell is literally #000000 (OLED-off) so it reads as punched out of the bezel, never a translucent HUD. One morphing element animates width+height+border-radius on a SwiftUI-style spring; closed is a true capsule (radius = height/2), open grows DOWNWARD from under the notch as one fluid blob. The four expanded regions (leading / trailing / center / bottom) ARE the layout grammar, with the bottom region holding the steer bar exactly as Apple puts the crucial controls there. Replace the in-repo concave NotchShape with the iOS capsule per the APPLE decision point. White-on-black only; no vibrancy, no gradient, no webfont.

**Palette:** `bg=#000000` `surface=#0A0A0A` `text=#FFFFFF` `muted=#FFFFFF99` `accent=#34C759` `border=#FFFFFF14`

**Type:** font `-apple-system, BlinkMacSystemFont, 'SF Pro Text', 'SF Pro Display', 'Helvetica Neue', Helvetica, Arial, sans-serif; numerals/timers use ui-rounded, 'SF Pro Rounded', system-ui`; sizes: compact label 15px/22 (SF Pro Text, the Infinum compact spec); expanded title 17px/22 (>=17 still Text, push to 20px Display for a hero); body activity 13px/18; secondary/timestamp 12px/16; tab chip 13px; weights: body/secondary 400; titles + emphasized values + active chip 600 (Apple uses Regular body, Semibold values)

**Radii:** closed capsule: border-radius 9999px (== height/2 at the 37px compact band, a TRUE pill, not a rounded rect — this is the APPLE decision-point choice over the repo's concave notch); expanded blob: 22px top corners, 32px bottom (large continuous ~44pt-family radius so the capsule->rect morph is one shape); chips + send button fully round (capsule); never cross-fade two shapes, morph the one element

**Spacing:** 8/16 grid (Apple expanded rhythm). Outer padding 16px; row gaps 8px; compact band hugs the capsule with the camera dead-zone reserved dead-center, everything vertically centered in the 37px height; expanded internal padding 8px; tab chips 9px x 4px; status dots exactly 6px

**Motion:** Shell morph = SwiftUI spring, Framer {type:'spring', stiffness:400, damping:30, mass:1} (the well-cited DI pair); for more Apple liveliness on OPEN drop dampingFraction toward 0.65 (repo openSpring is response:0.42/damping:0.80 — lower it). CSS-only fallback approximating the soft overshoot: cubic-bezier(0.34,1.56,0.64,1) over ~380ms. Option+Space reveal: cubic-bezier(0.32,0.72,0,1) over 280ms (Apple's snappy ease). Region content fades 200-300ms opacity while the shell runs the longer spring. Micro states (caret, send enable, dot color) 140ms ease-out. Closed box-shadow:none (part of the bezel); expanded box-shadow:0 4px 10px rgba(0,0,0,0.45) (matches the repo's open shadow). Respect prefers-reduced-motion: cut the spring to a 120ms opacity swap.

**Session view:** Compact closed = the bare 37px capsule (a tiny accent status dot leading, a session count trailing, dead-center reserved). On Option+Space it morphs open to ~408pt wide / up to 144pt (~192px) tall: this is the new-session composer. Leading region = a small BlitzOS agent glyph; trailing region = empty or a faint 'new'; center region UNDER the camera = title 'New session' 17px/600; BOTTOM region = the chat composer full-width (see chatBar). Auto-focus on open. No toolbar, no model picker.

**Process view:** Same morphed blob. Map to Apple's four regions: leading = agent glyph + 6px state dot; trailing = state value (e.g. a timer or step count, SF Rounded); center = agent title 17px/600 truncated; BOTTOM = the activity feed (13px/18 white-90, secondary lines 12px white-60) scrolling ABOVE the pinned steer bar. Tabs render as a 26px horizontal chip strip just under the center title (chip 13px, active 600 white, inactive 400 white-60, 6px dot); horizontal swipe switches tabs on the same spring (reuse the repo's DragGesture minimumDistance:24, width>height guard). Bottom region holds the steer bar exactly as Apple reserves the bottom row for crucial controls.

**Chat bar:** Pinned to the bottom region, full width. Single-row min-height 37px to echo the compact band; a wrapper holds a 1px rgba(255,255,255,0.08) border + #0A0A0A fill, radius 18px (capsule when single-row, swelling to ~14px rounded as it grows). Textarea: transparent, border:none, color #FFF, 15px/22, padding 10px 12px, overflow on the inner textarea (not overflow:clip on the rounded wrapper — the COMPOSER Chromium clip bug). Send = SF Symbol arrow.up.circle.fill 20px, align-self:end so it stays put as text grows; muted #FFFFFF99 when empty/disabled, accent #34C759 when draft.trim() is non-empty. Field-sizing:content primary path (Electron/Chromium 123+) with the scrollHeight reset-to-'auto' fallback — NOT the repo's reset-to-'42px' (chat.tsx L292). IME guard mandatory: bail on nativeEvent.isComposing || keyCode===229 before the Enter==send branch (chat.tsx L454 lacks it). Uncontrolled (defaultValue + ref.value on send) to keep native Cmd+Z and avoid re-rendering the feed.

**Borrow from:** native/island-helper/main.swift: keep the FIXED-window anti-flicker rule (animate the inner .island div, never the Electron BrowserWindow), the IslandModel tab/state machine, the DragGesture swipe (minimumDistance:24 + width>height), the dotColor() state->color map, and the bottom-pinned composer layout — but SWAP the concave NotchShape for the iOS capsule per the APPLE decision point. From widgets/system/chat.tsx borrow the .composer grid (grid-template-columns: minmax(0,1fr) 44px; align-items:end) and FIX its three flagged bugs (autogrow reset 'auto', IME guard, uncontrolled value). Spring numbers from APPLE (stiffness:400/damping:30) and animations.dev.

## rams — Less, But Better
**Philosophy:** Dieter Rams Principle 10 as a veto on every element: the resting island is ONLY a textarea + one send glyph (<=3 visible parts at rest). Achromatic warm-neutral greys and off-white from the Hue Atlas Braun palette, with EXACTLY ONE functional accent at a time and never two colors in one view (Braun monochrome discipline). Innovation lives in the interaction (notch overlay, swipe-between-agents, bottom steer), the skin stays Rams-quiet: flat fills, hairline 1px separators, one corner-radius family, strict 4/8 grid, no gradient/blur/glow. Signal colors are HONEST — red only for real errors/stops, yellow only for needs-input.

**Palette:** `bg=#1A1A18` `surface=#2E2E2C` `text=#F0EDE5` `muted=#8A8A87` `accent=#D4B018` `border=#FFFFFF14`

**Type:** font `-apple-system, BlinkMacSystemFont, 'SF Pro Text', 'Helvetica Neue', Helvetica, Arial, sans-serif (Principle 7 long-lasting: system font ages with macOS)`; sizes: composer input 14px/1.45; process/agent title 13px/1.4; body activity 13px/1.45; status & timestamps 11px/1.4 in muted; tab label 12px; weights: body/input/secondary 400; titles + active tab 600; letter-spacing 0, only -0.01em on titles

**Radii:** ONE family, concentric math (shell = inner + padding): outer shell 16px (echoes Sonoma window radius), inner input 10px with a 6px inset, chips/buttons 8px. Flat, no squircle theatrics. Hairline separators are pixel-snapped 1px (0.5px on @2x via transform).

**Spacing:** strict 4/8/12/16/24/32 grid, no arbitrary 13/17px values (Principle 8). Internal padding >=12px; >=8px between rows; status dots all exactly 6px; caret, send glyph and dots optically aligned to one baseline

**Motion:** Calm, never bounce. Swipe between tabs = a critically-to-slightly-underdamped spring stiffness 280 / damping 30 / mass 1 (settles, no cartoon overshoot). Option+Space reveal = cubic-bezier(0.32,0.72,0,1) over 280ms. All micro-state changes (focus, send enable, dot color) 120-160ms ease-out, never exceed ~300ms. Honor prefers-reduced-motion by DISABLING the swipe spring entirely (instant tab swap). Light variant flips shell to Braun White #F0EDE5 / text warm-black #1A1A18.

**Session view:** At rest the closed island is a quiet near-black #1A1A18 extension of the notch — no logo, no app name, no chrome buttons (Principle 5 unobtrusive). On Option+Space it opens to a single centered composer: a #2E2E2C input (radius 10px) on the #1A1A18 shell, one-line placeholder 'Start a BlitzOS session' in muted #8A8A87, one send glyph. Nothing else — no toolbar, no model dropdown, no attach icon (Principle 10). Generous padding 16px.

**Process view:** Tabs derived from real state, ordered by recency (Principle 4 self-explanatory): each tab = agent name 12px + a 6px status dot, active tab in the single accent #D4B018, inactive muted #8A8A87. No 'Tabs' label, no hamburger. Below: an honest activity feed (real tool-calls/steps as they happen, 13px/1.45 text, 11px muted timestamps) — no fake 'AI is working' theatre, no progress bar that doesn't track real progress. Spinner reflects true in-flight work only. Steer bar pinned at bottom. Signal colors used HONESTLY: red #C02820 only for genuine error/stop, yellow #D4B018 only for awaiting-input, muted grey idle.

**Chat bar:** Pinned bottom, MIRRORS the session composer exactly (Principle 8 consistency): same ~36px height, same 10px radius, same send glyph, so the two composers feel like one control across states. Wrapper holds the border (1px rgba(255,255,255,0.08)) + #2E2E2C fill; textarea transparent/borderless, 14px/1.45, vertical padding (36-21)/2 so single-row text is centered. Send = monochrome SF Symbol arrow.up.circle.fill, adopts the accent ONLY when the field is non-empty (else muted). Focus ring is restrained: a 1px accent-tinted border on :focus-within, no glow. Autogrow via field-sizing:content (cap ~120px). IME guard + uncontrolled value as in the COMPOSER findings.

**Borrow from:** Palette + discipline from RAMS (Hue Atlas hexes: Warm Black #1A1A18, Anthracite #2E2E2C, Braun White #F0EDE5, Braun Grey #8A8A87, Function Yellow #D4B018, Signal Red #C02820; the 'one accent, scarce, functional' rule). Layout skeleton + the consistency-across-composers idea from native/island-helper/main.swift (TabStrip 6px dot + capsule chip, MessageList + bottom input). The 280/30/1 spring and 280ms reveal from RAMS motion tokens. Composer correctness (field-sizing, IME guard, uncontrolled, wrapper-holds-border) from COMPOSER + chat.tsx (fixing its 42px-reset and missing IME guard).

## alcove — Alcove Glass
**Philosophy:** A faithful clone of the reverse-engineered Alcove tokens: a pure-#000 chassis with a continuous-corner (squircle) NotchShape, NOT a CSS border-radius — top corners tight to the screen edge, bottom corners large, so the panel reads as the notch physically swelling open. The exact extracted accent is teal #2DD4BF. Signature look = a vertical PROGRESSIVE/variable blur so the panel edges melt into the wallpaper, a white-8% top specular highlight, a black-25% outer border. Concentric corners (inner radius = outer minus frame inset). Interaction grammar is 1:1: tap-to-expand, HORIZONTAL swipe to change the active item (process tabs), vertical swipe to toggle open, velocity-tracked spring so flung swipes carry momentum, hover-to-expand after a ~0.25s dwell.

**Palette:** `bg=#000000` `surface=#1C1917` `text=#FEFFFF` `muted=#78716C` `accent=#2DD4BF` `border=#FFFFFF14`

**Type:** font `-apple-system, 'SF Pro Text', 'SF Pro Display', system-ui (Alcove bundles NO custom font); timer/clock uses font-variant-numeric: tabular-nums (binary uses monospacedDigit)`; sizes: compact strip glyphs 18px (Alcove's 18x18 inline icon); expanded artwork/agent tile 52px (Alcove's 52x52); title 13px/600; body 12px; secondary 11px in muted #78716C; chip 11px; weights: body 400 (textWeight=regular); titles + active SF Symbols 600 (selectedSymbolWeight=semibold); media/transport glyphs use the .fill variant

**Radii:** CONTINUOUS-corner squircle, not border-radius (Alcove's NotchShape + setCornerCurve(.continuous), per-corner radii). Collapsed: lower corners ~8px (matches the hardware notch bottom radius), top corners ~2px flush to the edge. Expanded: bottom corners ~28px continuous, top corners stay tight (~2px) so it grows OUT of the notch. In React approximate with an SVG/clip-path squircle 'd' path (a plain border-radius misses the bar). Corners concentric: inner content radius = outer minus the ~10px frame inset. Buttons = capsule (RoundedButtonStyle).

**Spacing:** ~10-14px content inset inside the expanded panel (Alcove NSEdgeInsets), ~6-8px around the collapsed strip. Container fill black @7.5% (#00000013), button fill white @5% (#FEFFFF0D) with a white-8% border (#FEFFFF14); modal/glass fill white @15% (#FEFFFF26).

**Motion:** Mass-spring-damper (Alcove's alignmentSpring{Stiffness,Damping,Mass} + velocity samples). Expand/collapse: snappy spring response ~0.35s / damping ~0.8; swipe-skip lighter response ~0.25s — and TRACK velocity so a flung swipe carries momentum (Framer: pass the drag velocity into the spring). Hover-to-expand gated by a ~0.25s dwell with a debounce (Alcove hoverDuration + _hoverDebounce) — reuse the repo's kCloseHoverDebounce pattern for the symmetric close. Signature visual motion = the variable blur fading in as the panel opens (DelayedBlurFade): CSS-fake it with backdrop-filter:blur(16px) masked by a mask-image:linear-gradient(top opaque -> bottom transparent) so the bottom edge melts into the wallpaper. Add haptics-equivalent feedback is N/A in web; keep per-state auto-hide timers + debounced inputs.

**Session view:** Collapsed = a thin pill hugging the notch, app glyph 18px LEFT of the physical notch, a tiny live indicator (spinner/dot) RIGHT (Alcove NotchLiveActivitiesView). Hover (after ~0.25s) or Option+Space grows it DOWNWARD with the progressive blur fading in into a rounded panel: the new-session composer. A 52px BlitzOS glyph tile + 'New session' title 13px/600 + the chat composer below. Top corners stay tight to the edge so it reads as the notch swelling.

**Process view:** NotchExpandedView equivalent: a full continuous-corner panel below the notch over the #000 chassis with the variable-blur bottom edge. Tab strip = one chip per agent (11px label + 6px state dot, active chip white-15% glass fill #FEFFFF26 + accent #2DD4BF dot/label, inactive muted #78716C). HORIZONTAL swipe switches the active tab (Alcove's swipe-to-skip-media mapped to swipe-between-processes) with the velocity-tracked spring; vertical swipe toggles open/closed. Activity feed in the body (12px text, 11px muted secondary), steer bar pinned at the bottom. Auto-hide on idle (Alcove _idleDuration) optional.

**Chat bar:** Pinned bottom inside the glass panel. Wrapper = button-style fill white @5% (#FEFFFF0D) + white-8% border (#FEFFFF14), capsule radius when single-row. Textarea transparent/borderless, color #FEFFFF, 13px/1.45, padding 10px 12px. Send = SF Symbol arrow.up.circle.fill, muted #78716C when empty, accent teal #2DD4BF when draft non-empty. Autogrow field-sizing:content (cap ~120px) with scrollHeight-reset-'auto' fallback; IME guard (isComposing||229); uncontrolled value (COMPOSER). overflow:hidden on the rounded wrapper (not clip).

**Borrow from:** ALCOVE for every token (exact extracted hex: chassis #000, surface stone-darker #1C1917, accent #2DD4BF, muted stone #78716C, button/container/modal fills #FEFFFF0D/#00000013/#FEFFFF26, white-8% borders) and the whole interaction grammar (tap-expand, horizontal-swipe-tabs, vertical-swipe-toggle, velocity spring, hover dwell, variable blur, continuous concentric corners). From native/island-helper/main.swift: the FIXED-window anti-flicker rule, the IslandModel + DragGesture swipe (it already does horizontal width>height), the hover-open + kCloseHoverDebounce pattern (Alcove's debounced hover), and the bottom-pinned composer. Composer correctness from COMPOSER + chat.tsx. Approximate Alcove's NotchShape with an SVG squircle clip-path since this is React/CSS not SwiftUI.

## expressive — Living Island
**Philosophy:** The crown-jewel direction: live, dynamic, agent-forward. Built on the current BlitzOS LIGHT tokens (tokens.css, sky-blue accent #38bdf8) but pushed expressive with per-agent accent tinting, an animated breathing status aura, and a goo/metaball morph so a spawning agent's tab visibly buds off the island. More color and motion than the others, but disciplined: motion always reflects REAL agent state (the OS doctrine — perception is dumb-but-rich, the agent decides significance), never decoration for its own sake. Frosted glass over the wallpaper, a soft accent glow on the active process, the live feel of an OS that's clearly thinking.

**Palette:** `bg=#0E1116` `surface=#FFFFFF14` `text=#F4F7FA` `muted=#A9B0B8` `accent=#38BDF8` `border=#FFFFFF1F`

**Type:** font `-apple-system, BlinkMacSystemFont, 'SF Pro Text', 'SF Pro Display', 'Geist', system-ui, sans-serif; big status numerals/timers in ui-rounded, 'SF Pro Rounded', system-ui with tabular-nums (friendly, alive)`; sizes: composer input 15px/1.45; agent title 14px/1.35; hero/center status 20px Display; body activity 13px/1.5; secondary/timestamp 11px/1.4; tab label 12px; weights: body 400; titles + active tab 600; hero status 600-700; -0.01em on titles

**Radii:** Friendly, larger family with a soft squircle feel: outer shell 24px (grows downward as one blob), inner input 14px, chips capsule, dots 6px. Concentric (shell = inner + 8px). Use a goo SVG filter (feGaussianBlur + feColorMatrix) on the tab/blob layer so a budding agent tab merges liquidly into the island (the APPLE metaball technique).

**Spacing:** 8px grid with generous breathing room. Outer padding 16-18px; row gaps 10px; chip padding 10px x 5px; input padding 12px 14px. The active process gets extra negative space + a glow halo so it reads as the focus.

**Motion:** Most alive of the five, all tied to real state. Shell/blob morph = lively spring, Framer {stiffness:380, damping:26, mass:1} (a touch of overshoot, the DI 'living organism' feel). Spawn = a metaball bud-off: the new tab grows from the island center via the goo filter over ~450ms. Active-agent status aura = a slow breathing pulse, reuse the repo's @keyframes breathe (scale .9->1.08, opacity .72->1) but ONLY while the agent is genuinely working (honest — stops when idle). Tab swipe = velocity-tracked spring response ~0.3s (flung swipes carry momentum, Alcove-style). Per-agent accent: each tab tints --blitz-accent to its own hue, the active process's glow + send button + dot all adopt it. Option+Space reveal cubic-bezier(0.34,1.4,0.64,1) over 320ms. Glass = backdrop-filter: blur(20px) saturate(1.6) with a top edge-highlight. prefers-reduced-motion: drop the breathing pulse, metaball, and overshoot; keep a 160ms fade.

**Session view:** Closed = a frosted capsule with a faint breathing accent dot (sky #38bdf8) over the wallpaper. Option+Space morphs it open with a soft overshoot into the new-session composer: a glass panel, a #FFFFFF14-on-glass input radius 14px, placeholder 'Start a BlitzOS session...', and a subtle animated accent glow behind the send button inviting the first message. Auto-focus.

**Process view:** The expressive heart. Tab strip = one capsule chip per agent, EACH tinted to its own accent hue; the active chip gets a soft colored glow halo + a breathing 6px state dot (green working / yellow waiting / coral error, pulsing only while truly active). Horizontal swipe between agents on the velocity spring; a spawning agent buds off via the goo metaball. Center can show a 20px SF Rounded hero status (e.g. step count / elapsed) for the active agent. Activity feed below (13px/1.5 #F4F7FA, 11px muted secondary) with real tool-calls streaming in, steer bar pinned at the bottom, its send button + caret tinted to the active agent's accent.

**Chat bar:** Pinned bottom, glass wrapper #FFFFFF14 + 1px rgba(255,255,255,0.12) border, radius 14px (capsule single-row, swelling as it grows). Textarea transparent/borderless, #F4F7FA, 15px/1.45, placeholder 'Steer this agent...'. Focus ring = a soft accent glow box-shadow 0 0 0 3.5px color-mix(in srgb, var(--agent-accent) 30%, transparent) on :focus-within (the active agent's hue). Send = arrow.up.circle.fill, muted #A9B0B8 when empty, the active agent's accent (animated glow) when draft non-empty, Stop square when in-flight. Autogrow field-sizing:content (cap ~140px) + scrollHeight-'auto' fallback; IME guard; UNCONTROLLED value (so the streaming feed never re-renders on keystroke — COMPOSER perf finding); overflow:hidden not clip on the rounded wrapper.
